# 01-编程基本功总览：为什么学 Linux 内核要先练好编程基本功

> 本部分是《Linux 内核与操作系统原理深入解析》的**配套动手训练**。它不重复讲解内核原理，而是把内核代码里反复用到的编程能力拆成可练习的"基本功"，让你从"能看懂"进阶到"写得出、调得动"。

## 1. 这本书的定位与常见缺口

本书系统讲解 Linux 内核（基于 3.10.29 / ARM）的架构、进程管理、内存管理、网络协议栈、文件系统与设备驱动。但"看懂原理"和"写得出来、调得动"之间，隔着一层**编程基本功**。

很多读者会卡在类似的地方：

- 看到内核里的 `struct list_head`、`container_of` 一头雾水——本质是 **C 指针与宏**功力不足；
- 读不懂 `set_bit()`、`atomic_t`、内存屏障——本质是 **计算机系统基础（位运算 / 内存模型）** 薄弱；
- 想动手验证某个原理，却不会用 `gdb`、`strace`、`/proc`——**工具链与调试**不熟；
- 照着抄了一段代码编译不过，或者跑起来段错误（segmentation fault）无从下手——**调试基本功**缺失。

本部分的目的，就是捅破这层窗户纸：用**可运行的最小代码 + 动手实验 + 调试实战**，把内核里用到的编程能力一项项练会。

## 2. 编程基本功能力地图（五大支柱）

| 支柱 | 核心能力 | 内核中的典型场景 | 对应训练章 |
|------|---------|----------------|-----------|
| **A. C 语言与系统编程** | 指针 / 内存模型 / 结构体 / 宏 / 位运算 / 系统调用 | `list_head`、`container_of`、`atomic_t`、文件 IO | 本章 02 |
| **B. 数据结构与算法** | 链表 / 红黑树 / 哈希 / 位图 / 队列 | `rbtree`、`hashtable`、`bitmap`、`kfifo` | 本章 03 |
| **C. 工具链与调试** | gcc / Makefile / gdb / valgrind / strace | 模块调试、性能分析、内存泄漏排查 | 本章 04 |
| **D. 计算机系统基础** | 数制 / 字节序 / 对齐 / 内存布局 / 中断 | 寄存器操作、MMIO、上下文切换 | 本章 05 |
| **E. 工程与规范** | 编码规范 / 防御性编程 / 自测试 | 内核 coding style、`BUG_ON`、kselftest | 贯穿各章 |

> 这五大支柱不是孤立的。例如读懂 `container_of` 需要 A（指针/宏）+ D（结构体内存布局）；排查一个并发 bug 需要 A + C + E。建议按 A→D→C→B→E 的顺序打底，再回到内核源码。

## 4. 怎么用这部分

1. **读原理时同步敲代码**：遇到 `list_head`、`htonl`、`mmap` 等，先来本部分找到对应支柱，把最小示例编译跑通。
2. **从错误中学**：本部分刻意保留了一些"会编译报错 / 会段错误"的反例，先自己诊断，再看解析。
3. **用工具验证直觉**：不要只"想当然"，用 `gdb` 看变量、用 `strace` 看系统调用、用 `/proc` 看运行时状态。
4. **循序渐进**：A、D 是地基，务必先练熟；B、C 是放大器；E 决定你能不能写出"像内核那样"的代码。

## 5. 环境准备

所有示例均可在普通 Linux 环境（物理机 / 虚拟机 / WSL / 开发板）编译运行：

```bash
# 编译器
gcc --version          # 或 clang
# 调试器
gdb --version
# 内存检查
valgrind --version     # 可选，用于内存泄漏实验
# 系统调用跟踪
strace -V              # 可选
# 确认字节序
lscpu | grep -i byte   # 查看当前机器大小端
```

示例约定：

- 所有 C 代码以 `// 示例 N` 开头，可独立保存为 `.c` 文件用 `gcc -std=c11 -Wall 示例.c -o 示例` 编译；
- 涉及内核 API（如 `list_head`）的示例会注明"用户态模拟"，即用标准 C 复现其思想，避免一上来就写内核模块劝退读者；
- 动手实验给出明确目标和验证方法，尽量不需要特殊硬件。
