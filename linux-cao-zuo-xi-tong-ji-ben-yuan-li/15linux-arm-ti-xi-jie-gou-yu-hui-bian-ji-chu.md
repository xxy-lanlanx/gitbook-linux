# 15-Linux ARM 体系结构与汇编基础

> 基于 Linux 3.10.29 内核、以 ARMv7-A（Cortex-A 系列）为主分析。本系列以 ARM 为分析对象，理解其体系结构与汇编基础，是读懂内核启动、异常向量、上下文切换的前提（补充 README 提到的「ARM 体系结构、汇编基础」）。

## 1. ARM 处理器模式

ARMv7 有若干**特权模式**，由 CPSR 的 M[4:0] 决定：

```text
User (usr)    用户态，最低特权（应用跑这里）
FIQ          快中断，高优先级
IRQ          普通中断
Supervisor(svc)  内核态/系统调用入口（上电与 svc 指令进入）
Abort (abt)   内存访问异常（缺页由此进入）
Undefined(und) 未定义指令
System (sys)  与 user 同寄存器但特权更高
```

- 应用跑在 **User**；内核跑在 **SVC**；
- 发生中断/异常时，硬件自动**切换模式并保存返回地址到对应模式的 LR**，跳到异常向量。

## 2. 寄存器

ARM 有 16 个 32 位通用寄存器 `r0–r15`，部分有专属用途：

```text
r0–r3    参数/返回值（调用者传前4个参数，r0 存返回）
r4–r11   被调用者保存的局部变量
r12 (ip) 内部过程调用临时寄存器
r13 (sp) 栈指针（每种模式有独立 sp）
r14 (lr) 链接寄存器（保存返回地址）
r15 (pc) 程序计数器
```

另有：
- **CPSR**：当前程序状态寄存器（模式、中断屏蔽位 I/F、条件码 N/Z/C/V）；
- **SPSR**：异常发生前的 CPSR 备份，用于返回时恢复。

> 内核上下文切换（见基本原理 12 章）本质就是保存/恢复这些寄存器 + 栈指针。

## 3. 异常向量与异常类别

ARM 异常向量表（内核中 `vector_stub` / `entry-armv.S`）位于固定地址（或 VBAR 指定）：

```text
0x00  Reset          复位（上电启动）
0x04  Undefined      未定义指令
0x08  SWI/SVC        软件中断（系统调用）
0x0C  Prefetch Abort 取指中止
0x10  Data Abort     数据中止（缺页中断由此进入！）
0x18  IRQ            普通中断
0x1C  FIQ            快速中断
```

用户程序执行 `svc #0` 触发 SVC 异常 → 进入内核 `sys_call` 路径（见基本原理 09 章系统调用）；访问未映射内存触发 Data Abort → 进入 `do_page_fault`（见内存管理 08 章）。

## 4. 汇编基础

### 4.1 指令格式与条件执行

ARM 指令可**带条件后缀**，仅当条件满足才执行（减少分支）：

```asm
  CMP  r0, #0          @ 比较 r0 与 0，设置标志
  MOVEQ r1, #1         @ 若相等(EQ) 则 r1=1
  MOVNE r1, #2         @ 若不等(NE) 则 r1=2
```

常见条件：`EQ`/`NE`/`GT`/`LT`/`CS` 等。

### 4.2 访存：LDR / STR（load/store 架构）

ARM 是 **load/store 架构**：运算指令只能操作寄存器，内存必须经 `ldr`/`str`：

```asm
  LDR  r0, [r1]        @ r0 = *(r1)          加载字
  LDR  r0, [r1, #4]    @ r0 = *(r1+4)       偏移
  LDR  r0, [r1, r2]    @ r0 = *(r1+r2)      寄存器偏移
  STR  r0, [r1]        @ *(r1) = r0         存储字
  LDRB/STRB            @ 字节
  LDM/STM              @ 批量加载/存储（常用于现场保护与恢复）
```

### 4.3 分支与函数调用

```asm
  B    label           @ 跳转（不保存返回）
  BL   func            @ 跳转到 func，并把返回地址写入 lr（调用）
  MOV  pc, lr          @ 返回（pc = lr）
```

### 4.4 ATPCS 调用约定（内核/用户约定）

- 参数：`r0–r3`，多余参数压栈；
- 返回值：`r0`（64 位用 `r0+r1`）；
- 被调用者需保存：`r4–r11`、`sp`、`lr`；
- 这些约定是汇编与 C 互调（如系统调用从汇编跳入 C）的基础。

## 5. 与内核启动的关系

上电后 Bootloader 把内核镜像搬入内存，跳到内核入口（解压、架构初始化）：
1. 早期汇编（`head.S`/`__lookup_processor_type`）：建立页表、开启 MMU；
2. 进入 C 的 `start_kernel()`（见基本原理 04 章内核初始化）；
3. 0 号进程 `swapper` 在 SVC 模式运行，后续 `fork` 出 1 号 `init`。

期间大量用汇编完成「关中断、切模式、建页表、保存现场」等 C 语言做不了的事。

## 6. 小结

ARM 通过**多特权模式 + CPSR/SPSR + 异常向量**实现特权隔离与异常响应；其 load/store 架构与 ATPCS 调用约定是汇编写出内核底层代码（启动、上下文切换、系统调用入口）的基础。掌握这些，才能顺着 README 规划的学习路径，真正读懂内核源码的「地基」。
