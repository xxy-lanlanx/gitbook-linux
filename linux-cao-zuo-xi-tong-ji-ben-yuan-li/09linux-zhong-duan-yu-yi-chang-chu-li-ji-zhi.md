# 09-Linux 中断与异常处理机制

> 中断是外设通知 CPU 的异步机制，异常是 CPU 执行指令时遇到的同步错误。二者是内核响应外部事件和内部错误的基石，也是理解驱动、调度、信号的关键。

## 1. 中断与异常的区别

| 特性 | 中断（Interrupt） | 异常（Exception） |
|------|------------------|------------------|
| 触发来源 | 外部设备（硬件） | CPU 内部（指令执行） |
| 同步性 | 异步（随时发生） | 同步（与指令流相关） |
| 典型例子 | 网卡收到数据、键盘按下 | 除零错误、缺页异常、非法指令 |
| 处理时机 | 当前指令执行完后 | 当前指令执行时或执行后 |
| 是否可屏蔽 | 部分可屏蔽（IF 标志） | 不可屏蔽（必须处理） |

> 在 Linux 内核中，中断和异常的处理流程高度统一，都通过**中断描述符表（IDT）**分发。

## 2. 中断描述符表（IDT）

x86 架构下，CPU 使用 **IDT（Interrupt Descriptor Table）** 来响应中断和异常。IDT 是一张表，每个表项（8 或 16 字节）描述了一个中断/异常的处理入口。

```
中断/异常号 → 查 IDT → 获取处理函数地址 → 跳转执行
```

### IDT 表项类型

| 类型 | 名称 | 用途 |
|------|------|------|
| 0 | 任务门 | 早期硬件任务切换，现已废弃 |
| 1 | 中断门 | 普通中断处理入口（最常用） |
| 2 | 陷阱门 | 异常处理入口（允许中断嵌套） |
| 3 | 系统门 | 用户态可触发的中断（如 `int 0x80`） |

Linux 内核初始化时，在 `arch/x86/kernel/idt.c` 中设置 IDT：

```c
static const __initconst struct idt_data early_idts[] = {
    INTG(X86_TRAP_DE,      divide_error),
    INTG(X86_TRAP_NMI,     nmi),
    INTG(X86_TRAP_PF,     page_fault),
    ...
};
```

## 3. 中断处理流程

### 3.1 硬件中断处理流程（以网卡为例）

```
网卡收到数据包
    │
    ▼
网卡通过 IRQ 线向中断控制器（APIC/IO-APIC）发信号
    │
    ▼
中断控制器把中断向量号送给 CPU
    │
    ▼
CPU 完成当前指令，查询 IDT，找到中断处理入口
    │
    ▼
保存现场（寄存器压栈）
    │
    ▼
执行中断处理函数（如 `e1000_interrupt`）
    │  • 关闭该中断（防止重入）
    │  • 读取设备状态，确认中断源
    │  • 把数据包放入接收队列
    │  • 触发软中断（NET_RX_SOFTIRQ）做后续处理
    │  • 恢复该中断
    │
    ▼
恢复现场，执行 `iret` 返回被中断程序
```

### 3.2 顶半部与底半部

中断处理要求**快进快出**，因为中断期间该 CPU 无法响应同级或更低优先级中断。Linux 把中断处理分为两部分：

| 部分 | 名称 | 职责 | 执行上下文 |
|------|------|------|----------|
| 顶半部（Top Half） | 硬中断 | 确认中断、读取设备、保存数据到缓冲区 | 硬中断上下文（不可睡眠） |
| 底半部（Bottom Half） | 软中断/任务队列 | 数据解析、协议栈处理、唤醒等待进程 | 软中断或进程上下文 |

**底半部的三种机制：**

1. **软中断（Softirq）**：编译时静态分配，优先级最高，同一类型可在多 CPU 并行执行。如 `NET_RX_SOFTIRQ`, `BLOCK_SOFTIRQ`。
2. **Tasklet**：基于软中断，可动态注册，同类型不能在不同 CPU 并行。
3. **工作队列（Workqueue）**：把任务挂到内核线程上执行，可以睡眠。适合耗时较长、需要等待 IO 的操作。

```c
/* 中断处理函数示例：顶半部 + tasklet 底半部 */
static irqreturn_t my_irq_handler(int irq, void *dev_id)
{
    /* 顶半部：快速处理 */
    disable_irq_nosync(irq);  /* 关闭本中断 */
    ...                       /* 读取设备状态 */
    tasklet_schedule(&my_tasklet);  /* 调度底半部 */
    return IRQ_HANDLED;
}

static void my_tasklet_handler(unsigned long data)
{
    /* 底半部：可以处理得更从容 */
    ...
}
```

## 4. 异常处理流程

### 4.1 缺页异常（Page Fault）

最常见的异常，当程序访问一个尚未映射或不在内存中的虚拟地址时触发：

```
程序访问地址 0x7f8a0000
    │
    ▼
MMU 查页表发现该页不在内存（PTE 的 Present 位为 0）
    │
    ▼
CPU 触发 #PF（Page Fault，异常号 14）
    │
    ▼
保存现场（cr2 寄存器记录故障地址）
    │
    ▼
调用 do_page_fault()
    │  • 检查地址合法性（是否在 vma 范围内）
    │  • 检查访问权限（是否可写/可执行）
    │  • 合法：分配物理页，建立映射，从磁盘换入（如果需要）
    │  • 非法：发送 SIGSEGV 给进程（Segmentation Fault）
    │
    ▼
恢复现场，重新执行触发异常的指令
```

> 缺页异常是 Linux 虚拟内存实现的核心机制。它实现了**按需分配（Demand Paging）** 和 **写时复制（Copy-on-Write）**。

### 4.2 其他常见异常

| 异常号 | 名称 | 触发条件 | 内核处理 |
|--------|------|---------|---------|
| 0 | 除零错误 | `div` / `idiv` 除数为 0 | 发送 SIGFPE 给进程 |
| 6 | 非法指令 | 遇到未定义的指令码 | 发送 SIGILL 给进程 |
| 13 | 通用保护故障 | 特权级违规、段越界 | 发送 SIGSEGV 或 SIGBUS |
| 14 | 缺页异常 | 访问未映射页 | 分配页/换入/发 SIGSEGV |

## 5. 中断与并发

### 5.1 中断嵌套

Linux 中断处理分为两层：

- **执行期间**：本地 CPU 关闭该中断线（防止同一中断重入），但其他中断线仍可响应。
- **通过 `irqflags` 控制**：`request_irq()` 时可以指定 `IRQF_SHARED`（共享中断线）或 `IRQF_DISABLED`（关闭所有中断，现已废弃）。

### 5.2 中断与自旋锁

中断处理函数运行在中断上下文，**不能睡眠**（不能调用 `kmalloc(GFP_KERNEL)`、`mutex_lock` 等）。如果中断和进程访问同一数据，需要使用**自旋锁（spinlock）**保护：

```c
spinlock_t lock;

/* 进程上下文 */
spin_lock_irq(&lock);   /* 关闭本地中断，获取锁 */
... /* 访问共享数据 */
spin_unlock_irq(&lock); /* 释放锁，恢复中断 */

/* 中断上下文 */
spin_lock(&lock);       /* 中断里不需要关中断 */
... /* 访问共享数据 */
spin_unlock(&lock);
```

> 为什么进程上下文用 `spin_lock_irq()` 而中断上下文用 `spin_lock()`？
> 因为中断可能打断进程，如果进程获取锁后被中断，中断也尝试获取同一把锁，就会死锁。所以在进程获取锁前需要先关闭本地中断。

## 6. 中断亲和性与负载均衡

现代多核系统中，中断可以绑定到特定 CPU：

```bash
# 查看网卡中断分布在哪些 CPU
cat /proc/interrupts

# 设置中断亲和性（把 eth0 的 IRQ 绑定到 CPU 0-3）
echo f > /proc/irq/123/smp_affinity
```

Linux 2.6 引入了 **IRQ 亲和性（IRQ Affinity）** 和 **IRQ 均衡**，允许：
- 把网卡中断分散到多个 CPU，提高并行收包能力。
- 把特定中断绑定到特定 CPU，减少缓存失效（NUMA 优化）。

## 7. 时钟中断与时间管理

**时钟中断（Timer Interrupt）** 是内核的"心跳"，由可编程间隔定时器（PIT）或本地 APIC 定时器产生。

- **默认频率**：100Hz ~ 1000Hz（`CONFIG_HZ`）。
- **作用**：
  - 更新 jiffies（内核全局时钟计数器）。
  - 检查时间片，触发进程调度（need_resched）。
  - 更新定时器队列，触发到期的内核定时器。
  - 更新资源统计（CPU 时间、进程统计）。

> 现代 Linux 引入了 **tickless（NO_HZ）** 模式，在空闲时关闭时钟中断，减少功耗，对笔记本电脑和服务器都有益。

## 8. 中断与系统调用的关系

| 场景 | 使用机制 | 触发方式 | 典型用途 |
|------|---------|---------|---------|
| 用户请求内核服务 | 系统调用 | `syscall` 指令 | 文件读写、进程创建 |
| 外设通知 CPU | 硬件中断 | 中断控制器 | 网卡收包、键盘输入 |
| CPU 执行错误 | 异常 | 指令执行 | 缺页、除零、非法指令 |
| 内核异步处理 | 软中断/任务队列 | 硬中断触发 | 协议栈处理、块设备完成 |

## 9. 小结

- **中断**是外部异步事件，**异常**是内部同步错误，二者都通过 IDT 分发。
- 中断处理分为**顶半部**（快速响应）和**底半部**（异步处理），避免中断处理过长。
- 底半部机制：软中断、tasklet、工作队列，根据场景选择。
- 中断上下文**不能睡眠**，与进程共享数据需要自旋锁保护。
- 时钟中断是内核的"心跳"，驱动调度、定时器、统计。
- 缺页异常是虚拟内存实现的关键，实现了按需分配和写时复制。
